\subsection{写作目的与读者模型}

本文档以\textbf{学术技术报告}体例撰写：在\textbf{不替代}仓库内 Markdown 长文证明（\fpath{Docs/POLICY.md}、\fpath{Docs/phases/}）的前提下，为 StructDB 的 C++ 实现提供\textbf{可复核}的\textbf{分层架构说明}、\textbf{模块边界}、\textbf{核心抽象（数据结构）}与\textbf{关键方法的行为语义及实现要点}。读者被假定为具备操作系统、存储引擎与并发基础的研究者或高级工程读者，需将本文与源码\textbf{对照阅读}。

\subsection{术语与约定}

\begin{description}
  \item[数据平面（Data Plane）] 指 \texttt{StorageEngine} 及其 WAL、MemTable、SST、Manifest、checkpoint、undo/redo 等持久化与检索路径。
  \item[控制平面（Control Plane）] 指 \texttt{Orchestration} 栈（Planner / Scheduler / GraphExecutor / Orchestrator）对维护任务（flush、checkpoint、L0 drain）的调度与预算。
  \item[门面（Facade）] 指 \texttt{structdb::facade::Engine}：对外暴露窄 KV 接口，并在写路径上叠加 WAL 队列预算与存储压力反馈。
  \item[嵌入层（Embed）] 指 \texttt{EmbedClient} 及 MDB 客户端：在引擎之上提供表逻辑、会话日志与脚本/REPL 语义。
  \item[读视图（Read View）] 由 \texttt{read\_max\_seq}（引擎 \texttt{kv\_get}）或 MDB 侧 \texttt{mdb\_storage\_read\_seq\_for\_script} 共同实现的\textbf{单调提交序列上界}裁剪。
\end{description}

  \item \textbf{工程真相源：}目标、选项与链接关系以 \fpath{CMakeLists.txt} 为准（第 \ref{sec:cmake-build} 节）。
  \item \textbf{语义与不变式真相源：}耐久、事务链、单写者等以 \fpath{Docs/POLICY.md} 及对应 PHASE 专文为准；本文给出\textbf{实现落点}（文件/类型/方法），并在明显处复述关键不变式\textbf{摘要}。

\end{itemize}

\subsection{两条正交控制流}

\paragraph{（A）交互式/脚本式请求流（L2$\rightarrow$L3$\rightarrow$L5）。}
MDB 文本行经 \texttt{mdb\_parse\_command\_line} 变为 \texttt{MdbParsedLine}，再经 \texttt{mdb\_dispatch\_execute\_line} 访问逻辑表状态并调用 \texttt{Engine::kv\_*} 或 \texttt{EmbedClient::submit}。该路径的\textbf{核心数据结构}为 \texttt{MdbParsedLine}、\texttt{MdbDispatchEnv}（第 \ref{sec:parsing-entry} 节）。

\paragraph{（B）引擎启动与维护图（L3$\rightarrow$L4$\rightarrow$L5）。}
\texttt{Engine::startup} 在存储 \texttt{open} 成功后构造预算器、调度器、执行器与 \texttt{Orchestrator}，注册 \texttt{noop}/\texttt{flush\_memtable}/\texttt{checkpoint}/\texttt{drain\_l0\_compaction} 等算子，并执行 \texttt{run\_default} 完成首次默认图。该路径的\textbf{核心数据结构}为 \texttt{OperatorNode}、\texttt{ExecutionPlan}、\texttt{ResourceBudget}（第 \ref{sec:mod-orchestration} 节）。

\subsection{写路径与读路径（摘要）}

\textbf{写路径（同步热路径）：} \texttt{Engine::kv\_put} $\rightarrow$（可选 24A 守卫）$\rightarrow$（可选异步队列）$\rightarrow$ \texttt{estimate\_put\_wal\_frame\_bytes} + \texttt{WalMutationBudget}（占用 \texttt{WalQueueDepth}）$\rightarrow$ \texttt{StorageEngine::put} $\rightarrow$（成功）\texttt{sync\_scheduler\_budget\_from\_storage\_pressure}。其中 \texttt{WalMutationBudget} 为 RAII，保证槽位配对释放（第 \ref{sec:impl-walkthrough} 节摘录）。

\textbf{读路径：} \texttt{Engine::kv\_get(..., read\_max\_seq)} 透传 \texttt{StorageEngine::get}；\texttt{read\_max\_seq} 与 \texttt{versioned\_kv} unwrap 规则共同决定可见版本（第 \ref{sec:mod-storage} 节与 \fpath{versioned\_kv.hpp}）。

\subsection{核心代码：运行时枢纽 \texttt{Engine::startup} 装配片段}

门面在存储 \texttt{open} 成功后注册维护算子并执行默认图；下列为 \fpath{engine.cpp} 中\textbf{编排装配}核心（中间大量 \texttt{set\_*} 已省略）。

\begin{lstlisting}[language=C++, numbers=left, firstnumber=174]
  auto budget = std::make_shared<scheduler::ResourceBudget>();
  auto sched = std::make_shared<scheduler::ExecutionScheduler>(budget);
  auto exec = std::make_shared<runtime::GraphExecutor>();
  exec->register_operator("noop", std::make_shared<NoopOperator>("noop"));
  exec->register_operator("flush_memtable", std::make_shared<FlushOperator>(storage_.get()));
  exec->register_operator("checkpoint", std::make_shared<CheckpointOperator>(storage_.get()));
  exec->register_operator("drain_l0_compaction",
                           std::make_shared<DrainL0CompactionOperator>(storage_.get(), drain_max));
  const bool defer_l0_compact = snap.l0_compact_defer_after_flush;
  orchestrator::PlanBuilder builder = [defer_l0_compact](std::uint64_t epoch) {
    if (defer_l0_compact) {
      return planner::ExecutionPlan::make_linear(
          epoch, std::vector<std::string>{"noop", "drain_l0_compaction"});
    }
    return planner::ExecutionPlan::make_linear(epoch, std::vector<std::string>{"noop"});
  };
  orch_ = std::make_shared<orchestrator::Orchestrator>(sched, exec, std::move(builder));
  orch_->set_before_graph_execute([this]() { sync_scheduler_budget_from_storage_pressure(); });
  // services_.register_singleton(...); orch_->run_default(&err);
\end{lstlisting}

\textbf{解释：}\texttt{FlushOperator}/\texttt{CheckpointOperator}/\texttt{DrainL0CompactionOperator} 将\textbf{控制平面节点名}映射为 \texttt{StorageEngine} 副作用；\texttt{PlanBuilder} 捕获 \texttt{l0\_compact\_defer\_after\_flush}，决定默认图是单 \texttt{noop} 还是 \texttt{noop$\rightarrow$drain\_l0\_compaction}；\texttt{before\_graph\_execute} 在每次图执行前刷新 WAL/合并槽预算（第 \ref{sec:mod-facade}、\ref{sec:mod-orchestration} 节详述）。
